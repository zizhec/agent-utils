# Lessons Skill

当用户调用 `/lessons` 时，执行以下步骤来回顾历史、总结错误并记录教训。

## 执行步骤

### 步骤 1: 读取现有 lessons.md

使用 Read 工具读取 `memory/lessons.md`，了解已经记录的错误，避免重复记录。

### 步骤 2: 搜索历史对话

使用 `episodic-memory:search-conversations` 技能搜索历史对话，查询以下关键词：
- 严重错误、失败、问题、bug、issue、mistake
- 不应该、错了、修复、解决
- 死循环、循环、重复、无限
- 重要教训、复盘、反思

### 步骤 3: 分析严重错误

从搜索结果中，识别真正的严重错误。特别注意以下模式：

**死循环模式**（高优先级）：
- 连续多次调用相同的工具
- 反复执行相同的命令序列没有进展
- 同样错误上反复尝试同样方案
- 工具调用陷入循环（A→B→A→B）
- 用户多次提醒但仍继续同样操作

**决策错误模式**：
- 错误的架构选择导致返工
- 跳过必要步骤（如未测试就提交）
- 对需求理解错误导致方向偏差

**执行错误模式**：
- 危险操作（如直接修改生产配置）
- 忽略警告或错误信息继续执行
- 未备份就进行破坏性操作

### 步骤 4: 格式化并写入

使用以下格式整理新发现的错误，并用 Edit 工具追加到 `memory/lessons.md` 文件末尾：

```markdown
## [YYYY-MM-DD] 错误标题

**场景**: 描述发生的上下文

**错误描述**: 具体做了什么错误的决策或操作

**后果**: 导致了什么问题

**解决方案**: 如何解决的

**预防措施**: 以后如何避免

---
```

### 步骤 5: 报告结果

向用户报告发现的错误数量和简要概述。

## 重要规则

1. 只记录真正的严重错误（影响进度、导致返工、设计失误、死循环等）
2. 确保是新的错误，不要重复记录已有的
3. 使用中文编写错误记录
4. 追加写入，不要覆盖原有内容
5. 发现死循环时记录：触发条件、循环模式、如何打破
