# 记忆 skills 对比

#### 1. memory (已安装)

 定位: 通用无限记忆系统

 核心特点:
 - 与内置 memory 并行不冲突
 - 用户自定义类别结构
 - 多层索引系统（Root → Category → Item）
 - 支持从内置 memory 同步

 存储结构:

 ```
   ~/memory/
   ├── config.md          # 系统配置
   ├── INDEX.md           # 根索引
   ├── [user-defined]/    # 用户自定义类别
   │   ├── INDEX.md
   │   └── {items}.md
   └── sync/              # 可选：从内置同步
 ```

 适用场景: 任何需要长期、结构化存储的数据

 ────────────────────────────────────────────────────────────────────────────────

 #### 2. remember (刚安装)

 定位: 智能筛选和组织记忆

 核心特点:
 - 筛选什么值得记住（高/中/低价值）
 - 按功能组织而非按内容
 - 记忆卫生: 日期、来源、置信度
 - 主动修剪: 30天归档、矛盾处理

 存储结构:

 ```
   memory/
   ├── commitments.md     # 承诺、截止日期
   ├── preferences.md     # 偏好、风格、边界
   ├── corrections.md     # 错误不再重复
   ├── decisions.md       # 决策记录
   ├── relationships.md   # 人物、角色
   └── contexts/          # 按项目/客户
 ```

 价值判断框架:

 | 价值级别 | 内容 | 处理方式 |
 |---------|---------|---------|
 | 高 | 承诺、纠正、偏好、关系 | 永久保存 |
 | 中 | 项目状态、领域教训、决策 | 定期审查 |
 | 低 | 一次性问题、易重构事实 | 不保存 |

 适用场景: 需要智能筛选和组织记忆，避免信息过载

 ────────────────────────────────────────────────────────────────────────────────

 #### 3. learn (刚安装)

 定位: 结构化学习跟踪

 核心特点:
 - 间隔重复 (Spaced Repetition)
 - 主动回忆 (Active Recall)
 - 掌握度验证 (5题中答对4题)
 - 学习计划制定

 存储结构:

 ```
   ~/learn/
   ├── topics/
   │   └── {topic}/
   │       ├── concepts.json   # 概念 + SR 计划
   │       ├── notes.md        # 学习笔记
   │       └── progress.md     # 掌握度跟踪
   ├── reviews/
   │   └── due.json            # 待复习队列
   └── config.json             # 偏好设置
 ```

 工作流程:

 ```
   Goal → Plan → Study → Practice → Verify → Review
 ```

 适用场景: 学习新技能、知识管理

 ────────────────────────────────────────────────────────────────────────────────

 #### 4. decide (刚安装)

 定位: 决策模式学习

 核心特点:
 - 学习决策模式而非执行偏好
 - 保守默认: 重大决策先问人
 - 组件化决策: 问题 + 影响因素 + 选择
 - 模式推广: 重复3次后询问确认

 存储结构:

 ```
   ~/decide/
   ├── memory.md        # 决策规则、边界
   ├── decisions.md     # 重大决策记录
   └── domains/         # 领域特定规则
 ```

 决策流程:
 1. 框定问题
 2. 收集影响因素
 3. 读取 memory → domains → decisions
 4. 仅当完全匹配时复用
 5. 否则询问并记录

 适用场景: 需要学习决策模式、避免重复询问相同类型决策

 ────────────────────────────────────────────────────────────────────────────────

 #### 5. self-improving (刚安装)

 定位: 自我反思 + 持续改进

 核心特点:
 - 三层存储: HOT (≤100行) / WARM (按需) / COLD (归档)
 - 自动升降级: 3次使用→HOT，30天未用→WARM，90天→COLD
 - 自我反思: 完成任务后评估
 - 心跳维护: 定期回顾和整理

 存储结构:

 ```
   ~/self-improving/
   ├── memory.md          # HOT: 常用规则
   ├── index.md           # 主题索引
   ├── heartbeat-state.md # 维护状态
   ├── projects/          # 项目特定
   ├── domains/           # 领域特定
   ├── archive/           # COLD: 归档
   └── corrections.md     # 最近50次纠正
 ```

 学习信号:
 - 纠正: "No, that's not right..."
 - 偏好: "I like when you..."
 - 模式: 重复3次的指令

 适用场景: 持续改进工作质量、学习用户偏好

 

 #### 🔍 cognitive-memory 详细分析

 定位: 类人认知架构记忆系统（最复杂、最全面）

 核心特点:

  |特性 | 说明 |
 |---------|---------|
 | 四存储模型 | Episodic（情景）+ Semantic（语义）+ Procedural（程序）+ Core（核心） |
 | 触发系统 | 自然语言触发（"remember", "forget", "reflect"） |
 | 衰减模型 | 指数衰减 + 访问频率 + 类型权重 |
 | 反思引擎 | 睡眠时反思 + 哲学自我审视 |
 | 知识图谱 | 实体关系图 |
 | 多Agent支持 | 共享读取、门控写入 |
 | 审计追踪 | Git + audit.log |
 | 身份进化 | Self-Image 随反思进化 |
 | 奖励机制 | Token 奖励请求系统 |




 存储结构:

 ```
   workspace/
   ├── MEMORY.md                    # 核心记忆 (~3K tokens)
   ├── IDENTITY.md                  # 事实 + 自我形象 + 自我意识日志
   ├── SOUL.md                      # 价值观、原则、承诺、边界
   └── memory/
       ├── episodes/                # 情景记忆：YYYY-MM-DD.md
       ├── graph/                   # 语义知识图谱
       │   ├── index.md             # 实体注册 + 边
       │   ├── entities/            # 每个实体一个文件
       │   └── relations.md         # 边类型定义
       ├── procedures/              # 程序记忆（工作流）
       ├── vault/                   # 保险库记忆（永不衰减）
       └── meta/
           ├── decay-scores.json    # 衰减分数
           ├── reflection-log.md    # 反思摘要
           ├── reflections/         # 完整反思档案
           ├── reward-log.md        # 奖励结果
           ├── rewards/             # 完整奖励请求
           ├── pending-reflection.md
           ├── pending-memories.md
           ├── evolution.md         # 进化追踪
           └── audit.log
 ```

 衰减模型:

 ```
   relevance(t) = base × e^(-0.03 × days) × log2(access_count + 1) × type_weight
 ```

|分数 | 状态 | 行为 |
|---------|---------|---------|
| 1.0–0.5 | Active | 完全可搜索 |
| 0.5–0.2 | Fading | 降低优先级 |
| 0.2–0.05 | Dormant | 仅显式搜索 |
| < 0.05 | Archived | 隐藏 |







 类型权重: core=1.5, episodic=0.8, semantic=1.2, procedural=1.0, vault=∞

 反思流程:
 1. 确认触发 - 用户说 "reflect" 或 "going to sleep"
 2. 请求 Token - 申请反思资源
 3. 执行反思 - 内部独白（5-8个元素）
 4. 记录结果 - 更新 reflection-log, reward-log, IDENTITY.md

 ### 🤔 关键发现

 #### 1. 分工明确

 每个 skill 解决不同层面的记忆问题：
 - memory: 存储基础设施
 - remember: 存储什么（筛选）
 - learn: 如何学习知识
 - decide: 如何做决策
 - self-improving: 如何持续改进

 #### 2. 互补关系

 ```
   remember 筛选 → memory 存储
   learn 学习知识 → self-improving 改进方法
   decide 学习决策 → self-improving 反思结果
 ```

 #### 3. 设计哲学差异

 - memory: "存储一切，用户组织"
 - remember: "只存重要的，智能筛选"
 - learn/decide/self-improving: "特定场景，专门优化"

 ### 🎯 关键洞察

 #### 1. 认知层级

 ```
   Level 1: 存储（memory）
       ↓
   Level 2: 筛选（remember）
       ↓
   Level 3: 学习（learn, decide）
       ↓
   Level 4: 改进（self-improving）
       ↓
   Level 5: 认知（cognitive-memory）← 最全面
 ```

 #### 2. cognitive-memory 的独特之处

 - 唯一有哲学自我审视
 - 唯一有 Token 奖励机制
 - 唯一有四存储模型
 - 唯一有知识图谱
 - 唯一有衰减模型
 - 唯一有身份进化

 #### 3. 使用建议

 | 需求 | 推荐 Skill |
 |-----------|----------------|
 | 简单存储 | memory |
 | 筛选重要信息 | remember |
 | 学习知识 | learn |
 | 学习决策 | decide |
 | 持续改进 | self-improving |
 | 项目上下文 | Nemp-memory |
 | 全面认知架构 | cognitive-memory |
